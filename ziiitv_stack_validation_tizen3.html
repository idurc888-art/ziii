⚠ Hardening
Parser M3U8
Playlist IPTV — precisa ser defensivo
▶
Lib recomendada
iptv-playlist-parser
Problema real
M3U8 de IPTV são sujos
Encoding comum
UTF-8 + Windows-1252
Tamanho típico
500KB – 5MB
Normalizar \r\n → \n antes de processar. Windows line endings quebram o parser em TVs.
Usar regex com optional chaining (?.[1]) — campos como group-title podem estar ausentes.
Processar em chunks ou com requestIdleCallback para não travar a UI em listas grandes.
Cachear no IndexedDB após primeiro parse — não reproces­sar na cada navegação.
⚠ Risco alto
EPG via XMLTV
Guia de programação — maior ponto de falha
▶
Tamanho típico
50 MB – 300 MB
RAM de TV (2017)
512 MB total
Parse completo?
❌ Inviável na TV
Estratégia certa
Sob demanda + cache
Nunca fazer fetch e parse completo do XMLTV na TV. 300 MB vai travar ou crashar o app.
Carregar EPG só do canal atual + próximas 2h. Buscar próximo canal só quando o usuário navegar.
Usar Cache API com TTL de 1h. XMLTV não muda a cada segundo — reutilizar o parse.
Considerar proxy backend leve que pre-processa o XMLTV e entrega JSON por canal — elimina o problema.
